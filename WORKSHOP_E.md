# 各地啟動 Onboarding 工作坊
### TODO-WORKSHOP-15（各地區複製場）

> **這場不是設計討論，是執行複製。**  
> 場次 A/B/C/D 定義了系統如何運作；這場的目的是讓每一個新地區的幹部，在半天內能夠獨立啟動當地的第一次作業。  
> 每個地區辦一次，由已完成訓練的幹部或設計者帶領。

---

## 基本資訊

| 項目 | 內容 |
|------|------|
| **時間** | 半天，建議 09:00–12:30（3.5 小時） |
| **地點** | 當地有投影設備的空間（分會辦公室或社區中心） |
| **人數** | 5–8 人（當地幹部 + 核心志工） |
| **前置條件** | 場次 D（人力/組織）已完成；系統帳號已開通 |
| **主持人** | 已受訓的幹部或設計者（不需要再次決策，只要教學） |

---

## 這場不討論什麼

- ❌ 不重新討論表單欄位（場次 A 已定）
- ❌ 不重新討論授權層級（場次 B 已定）
- ❌ 不重新設計流程（場次 C 已定）
- ❌ 不更動組織架構（場次 D 已定）

如果當地有特殊情況需要調整規則，記錄下來，帶回總部討論，不在本場處理。

---

## 產出物

1. **當地帳號全部開通並登入**
2. **第一個測試個案建立完成**
3. **當地幹部知道遇到問題要找誰**
4. **本地特殊情況清單**（帶回總部討論用）

---

## 議程

### 09:00　帳號確認（20 分鐘）
- 每個人打開系統，確認能登入
- 確認角色權限正確（組長看到組長介面、幹部看到幹部介面）
- 有問題：當場解決，解決不了記錄下來

---

### 09:20　系統導覽（40 分鐘）
主持人帶著大家走一遍主要功能，不深入，只認路：

1. **個案管理**：在哪裡、長什麼樣子（5 分鐘）
2. **志工管理 + 技能標籤**：怎麼找人（5 分鐘）
3. **任務派遣**：怎麼建任務、怎麼指派（10 分鐘）
4. **評估表單**：手機怎麼填（10 分鐘）
5. **援助紀錄 + 核准流程**：錢怎麼批（10 分鐘）

---

### 10:00　動手練習（60 分鐘）
每個人用自己的帳號，完成以下操作（主持人在旁輔助）：

| 操作 | 完成 |
|------|------|
| 建立一個測試個案（用假地址）| ☐ |
| 填寫損害評估表單（手機）| ☐ |
| 建立一個任務並指派給旁邊的人 | ☐ |
| 接受任務並回報完成 | ☐ |
| 建立援助紀錄（金額 1 元，測試用）| ☐ |
| 核准援助紀錄（幹部帳號）| ☐ |
| 把測試個案標記為「測試，請忽略」| ☐ |

---

### 11:00　本地特殊情況盤點（20 分鐘）
主持人問，當地幹部回答：

- 你們地區有什麼特殊情況，覺得系統現在沒考慮到的？
- 語言問題？（台語/客語/原住民族語）
- 地形問題？（山區/離島/偏遠）
- 組織結構特殊？（只有 2 個幹部、兼任很多角色）

**設計者記錄，不當場解決，帶回。**

---

### 11:20　緊急聯絡與支援管道（10 分鐘）
確認每個人知道：
- 系統出問題，打給誰？（姓名 + 電話）
- 規則有疑問，問誰？（姓名 + 聯絡方式）
- 帳號問題，找誰？

發給每人一張「緊急聯絡卡」（A5，實體或數位）。

---

### 11:30　第一個真實任務預排（30 分鐘）
如果當地近期有已知案件或即將到來的活動：

- 在系統裡建立第一個真實個案（不是測試）
- 確認負責人已指派
- 確認系統資料是否正確

如果目前沒有真實案件，跳過這段，提早結束。

---

### 12:00　收尾（30 分鐘）
- 確認所有人帳號正常
- 回收「本地特殊情況清單」
- 說明後續支援方式（Line 群組？Email？電話？）

---

## 主持人準備清單

- [ ] 所有帳號在工作坊前 2 天開通（讓當地幹部先試登入）
- [ ] 準備「緊急聯絡卡」（含支援窗口資訊）
- [ ] 準備「本地特殊情況記錄表」（空白 A4）
- [ ] 確認當地投影設備可連筆電
- [ ] 準備備用網路（手機熱點）

---

| 評估是否需要調整系統規則 | 設計者 | +14 天 |
| 確認當地已完成第一個真實任務 | 當地幹部 | +30 天 |
| 更新 TODO_WORKSHOP.md（標記 15 完成）| 設計者 | 每場結束後 |
