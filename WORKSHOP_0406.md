## 產出物（工作坊結束時手上要有）

1. **數位化評估表單草稿**——欄位清單 + 必填/選填標記 + 照片拍攝角度說明
2. **三種修繕類型的里程碑清單**——屋頂 / 水電 / 結構輕損，各節點驗收條件
3. **費用估算邏輯描述**——不需精確數字，只需「怎麼算」的邏輯框架

## 議程

### 09:00　破冰（15 分鐘）
每位師伯說一個印象最深的修繕案例，用 2–3 句話描述。

**主持人引導問題：**
- 那個案子最難的地方是什麼？
- 你怎麼決定要怎麼修？

> 目的：讓師伯進入思考狀態，設計者聽到真實現場情境。

---

### 09:15　現有紀錄盤點（30 分鐘）
請師伯把帶來的所有紀錄格式攤開，逐一說明。

**主持人引導問題：**
- 你評估一戶通常記哪些東西？
- 什麼情況會影響你的判斷，但你不一定會寫下來？
- 有沒有覺得「這個一定要記」但現在沒地方寫的資訊？

**設計者同步做：** 把提到的所有欄位名稱寫在白板左欄，不分類、不過濾，先全部列出。

> 產出：欄位清單草稿（原始版，未整理）

---

### 09:45　損害分類標準化（45 分鐘）
**目標：** 讓系統知道怎麼分類損害，後續的里程碑和費用才有對應的基準。

**主持人引導問題：**
- 你看到一棟房子，什麼情況叫「輕損」？什麼叫「中損」？什麼叫「重損」？
- 結構損傷、屋頂損傷、水電損傷，判斷方式一樣嗎？
- 有沒有「看起來輕損但其實很麻煩」的情況？

**白板工具：** 畫一個 4×4 矩陣（損害部位 × 嚴重程度），讓師伯逐格填入判斷標準。

| | 輕損 | 中損 | 重損 | 全損 |
|---|---|---|---|---|
| **結構** | | | | |
| **屋頂** | | | | |
| **水電** | | | | |
| **外牆** | | | | |

> 產出：損害類型 × 嚴重程度的判斷標準矩陣

### 10:30　休息（10 分鐘）

### 10:40　修繕里程碑討論（40 分鐘）
**目標：** 定義各種修繕類型的驗收節點，系統才能追蹤進度。

**主持人說明：**
> 「我們要為最常見的修繕類型，定義幾個『檢查點』。每個檢查點就是一個里程碑，師伯到場確認後才能進入下一階段。」

**討論三種類型（各約 12 分鐘）：**

**屋頂修繕：**
- 幾個驗收節點？（例：開工前現勘 → 防水層完成 → 瓦片/鐵皮完成 → 收尾清潔）
- 每個節點要確認什麼？要拍什麼照片？

**水電修繕：**
- 幾個驗收節點？（例：斷電/斷水確認 → 管線更換完成 → 通電/通水測試 → 完工）
- 水電有沒有需要請領政府執照的情況？

**結構輕損：**
- 幾個驗收節點？（例：裂縫現勘 → 補強施工 → 強度測試 → 完工）
- 什麼情況需要升級請結構技師？

> 產出：3 種修繕類型的里程碑清單（每種 3–5 個節點）

---

### 11:20　費用估算方式（30 分鐘）
**目標：** 了解師伯怎麼估費用，系統才知道要設計什麼欄位。

**主持人引導問題：**
- 你估一個屋頂修繕，腦子裡怎麼算的？先算材料還是先算工資？
- 面積怎麼量？有沒有快速估算方式？
- 台北和花蓮的工資差多少？材料呢？
- 有沒有「這個我估不準，需要廠商報價」的情況？

**設計者同步做：** 記錄估算流程（不是數字，是步驟），例如：
```
現場丈量面積 → 計算材料用量（依損害程度）
→ 材料單價（參考當地行情）→ 工資（面積 × 日工資）
→ 加總 → 超過 XX 萬需廠商複估
```

> 產出：費用估算邏輯流程文字描述

---

### 11:50　KoboToolbox 試填（30 分鐘）
**目標：** 讓師伯用手機實際操作，找出不合理的地方。

**主持人說明：**
> 「我把剛才討論的內容做成了一份初版表單，請大家拿手機填看看，假設你現在在某個案家門口做評估。」

**觀察重點：**
- 師伯在哪個欄位卡住了？
- 有沒有欄位是「這樣問不對」或「少了某個選項」？
- 手機填起來方不方便？有沒有要用相機的地方？

**設計者同步做：** 在旁邊用筆記本記錄每一個「卡住點」，不解釋、不辯護，先全部記下來。

> 產出：師伯的直接反應清單（「這個欄位不對」「少了什麼」「這個選項沒有涵蓋 XX 情況」）

---

### 12:20　收尾 + 下一步確認（10 分鐘）

**確認事項：**
1. 三份產出物各由誰整理成文件？（建議：設計者整理初稿，師伯中一位 review）
2. 下次修訂時間？（建議 2 週內）
3. 哪些問題今天沒有答案，需要再問誰？
