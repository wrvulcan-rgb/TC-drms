## 議程

### 09:00　破冰（20 分鐘）
每位參與者說：你帶過最大的一次任務，需要哪些不同技能的志工？

**主持人引導問題：**
- 最難找到的技能是什麼？
- 臨時調人的時候你怎麼找？有沒有方法？

---

### 09:20　志工能力盤點（60 分鐘）
**目標：** 定義系統能辨識、能搜尋的技能標籤。

**主持人引導問題：**
- 你找人的時候，你會問「有沒有會 __ 的人」——那個空格通常填什麼？
- 修繕類：水電、木工、油漆、鐵工、磁磚、結構——還有什麼？
- 通用類：開車、翻譯、醫護、攝影、文書——還有什麼？
- 有沒有技能是「要有執照才能做」的？（水電師傅、護理師）
- 有沒有技能是「不是技術，但很重要」的？（方言溝通、熟悉地形）

**白板工具：** 便利貼分類牆
- 每人寫出 5–10 個技能名詞（一張一個）
- 貼到白板，設計者引導合併相似標籤
- 目標：最終清單不超過 20 個標籤

**設計者確認：**
- 每個標籤用系統能處理的方式呈現（不能有模糊詞）
- 哪些需要「等級」標記（初級/中級/高級）？
- 哪些是「有/無」二元的？

> 產出：志工技能標籤清單（≤20 個）

---

### 10:20　動態編組規則（50 分鐘）
**目標：** 定義 TeamSession（動態編組）的生命週期與規則。

**主持人說明：**
> 「系統裡，每次出任務都會建立一個『動態編組』（TeamSession），不是固定的班組。今天我們要定義怎麼建立這個編組。」

**主持人引導問題：**
- 一個任務通常需要幾個人？最少幾個可以出發？
- 誰有權建立一個編組？（只有幹部？還是志工也可以自發？）
- 怎麼決定誰加入？（自願？指定？系統媒合？）
- 有沒有「這個人一定要在」的核心角色？（例：一定要有持照水電工）
- 一個人可以同時在幾個編組？

**白板工具：** 畫出 TeamSession 生命週期

```
[建立] → [招募成員] → [集合確認] → [執行任務] → [收班] → [解散]
           ↓
       [成員拒絕] → [重新招募]
```

讓幹部在每個箭頭填入條件與觸發者。

> 產出：動態編組建立規則文件

---

### 11:10　梯次排班規則（40 分鐘）
**目標：** 定義梯次（Tier）排班的時間與交接邏輯。

**主持人引導問題：**
- 你們現在怎麼排班？一個梯次多長？（半天？全天？多天？）
- 梯次結束時，未完成的任務怎麼辦？誰負責交接？
- 有沒有跨梯次都要有人的「值勤」角色？
- 遠地作業（需要住宿）和本地作業的排班規則一樣嗎？
- 梯次人數的上下限怎麼決定？

**設計者記錄：**
- 梯次時間結構（例：上午 08:00–12:00 / 下午 13:00–17:00 / 全天 / 多日）
- 交接必要文件清單
- 排班系統需要的欄位

> 產出：梯次排班規則文件 + 交接清單

### 12:50　幹部角色與權限（60 分鐘）
**目標：** 定義系統內每個角色能做什麼，不能做什麼。

**主持人說明：**
> 「系統裡，不同角色看到的介面和能操作的功能不一樣。我們要定義哪些角色存在，每個角色的權限範圍。」

**先列出現有角色（讓幹部補充）：**
- 一般志工
- 組長（班長）
- 地區幹部
- 財務幹部
- 地區主委
- 系統管理員
- 觀察員（唯讀）

**白板工具：** 角色 × 功能矩陣

| 功能 | 一般志工 | 組長 | 地區幹部 | 財務 | 主委 |
|------|---------|------|---------|------|------|
| 建立個案 | | | | | |
| 審核個案 | | | | | |
| 建立任務 | | | | | |
| 核准援助金 | | | | | |
| 查看所有個案 | | | | | |
| 管理志工資料 | | | | | |
| 查看財務報表 | | | | | |
| 系統設定 | | | | | |

**填入規則：** ✅ 可以 / ❌ 不行 / 🔒 需要申請

**主持人引導問題：**
- 有沒有「只有本地區能看」的個案？跨區協調時怎麼開放？
- 志工能看到其他志工的聯絡資料嗎？（隱私問題）
- 幹部離職或換人時，權限怎麼轉移？

> 產出：角色權限矩陣

---

### 13:50　跨地區協調機制（40 分鐘）
**目標：** 定義大型災害時，多個地區慈濟協作的規則。

**主持人引導問題：**
- 有沒有一個地區的案件量超過自己能處理的情況？怎麼求援？
- 外區志工來支援，在系統裡算哪個地區的人？
- 不同地區的主委，誰有最終決策權？
- 跨區資源（車輛/物資）的調度，誰能決定？
- 跨區協作結束後，資料怎麼歸屬？

**設計者記錄：**
- 跨區協作觸發條件（達到什麼狀況才啟動？）
- 臨時指揮權的歸屬規則
- 系統層面：一個帳號是否能跨區操作？

> 產出：跨地區協調規則文件

### 14:45　新志工 Onboarding 流程（45 分鐘）
**目標：** 定義新志工從「我想加入」到「第一次出任務」的完整流程。

**主持人引導問題：**
- 現在新志工怎麼加入？誰審核？多久？
- 加入後要接受什麼訓練才能出任務？（安全規定/系統操作/特定技能）
- 有沒有「試用期」或「見習」的概念？
- 新志工第一次出任務，旁邊一定要有資深志工嗎？
- 什麼情況下志工資格會被暫停或取消？

**設計者記錄：** 繪製 onboarding 流程圖

```
[申請加入] → [基本資料填寫] → [幹部審核] → [通過]
                                               ↓
                              [安全說明確認] → [系統操作訓練]
                                               ↓
                              [填寫技能標籤] → [等待分配任務]
                                               ↓
                              [第一次任務] → [資深志工陪同]
                                               ↓
                              [任務完成] → [正式志工]
```

> 產出：Onboarding 流程圖 + 每個步驟的負責人與時間

---

**觀察重點：**
- 技能標籤在哪裡填寫？容易找到嗎？
- 組長怎麼搜尋有特定技能的志工？
- 梯次排班介面在哪裡？
- 權限設定介面是否清楚？

> 產出：介面問題清單
